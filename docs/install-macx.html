Qtstalker: Installing on Mac OS X

Requirements and dependencies

  • Mac OS X (Leopard 10.5 or Snow Leopard 10.6 or Lion 10.7).
  • Apple Xcode Tools. To build Qtstalker from sources.
  • Dependencies: Qt and Qwt and TA-Lib

Installation

See the general Installation instructions for overview, requirements, dependencies, and tips. Note that these Mac instructions are deliberately terse because the main installation instructions have the detail.

There are other build systems and IDEs, however this works from the command-line with minimal changes to Qtstalker sources.

The dependencies are easy to install from their source. You might be able to install these using package managers. However use the examples of each dependency to ensure proper installation, before complaining about Qtstalker.

Install TA-Lib

Install TA-Lib 0.4.0 from source. 

./configure
make
sudo make install

This will install TA-Lib at /usr/local ... See important notes about ensuring proper installation and configuration if at a non-standard location.

Install Qt

Install Qt 4.6 or newer from source.

For Lion and Leopard use the default configuration: 

./configure

For Snow Leopard (or perhaps it was Qt-4.6 on SL) this configuration was needed: 

./configure -cocoa -arch x86_64

See important notes about ensuring proper installation and adding to your $PATH environment variable to ensure that 'qmake' etc can be found.

Install Qwt

Install Qwt 5.2 from source.

See important notes about ensuring proper installation and adding to your $DYLD_LIBRARY_PATH environment variable to ensure that its shared library can be found.

Building Qtstalker

Prepare for compiling Qtstalker:

By default Qt and Qwt are installed at a non-standard location (which is difficult for Qtstalker to automatically guess). So edit the qtstalker.config file to add the following lines (using whatever relevant version numbers): 

INCLUDEPATH += /usr/local/Trolltech/Qt-4.6.3/include/Qt
LIBS += -L/usr/local/Trolltech/Qt-4.6.3/lib
INCLUDEPATH += /usr/local/qwt-5.2.1/include
LIBS += -L/usr/local/qwt-5.2.1/lib

For Snow Leopard, edit our "configure" file to tweak the "qmake" invocation to include "CONFIG += qt x86_64". See the Qt notes above, perhaps this was just with Qt4.6 on SL. For Lion, no changes are needed.

Now compile Qtstalker: 

./configure
make
sudo make install

Post-install steps

At runtime, the "qtstalker" program will need to be discoverable, so create symbolic links: 

cd /usr/local/bin
sudo ln -s qtstalker.app/Contents/MacOS/qtstalker qtstalker

Configure the Finder to enable access to Qtstalker scripts. Unless you modified the installation locations using qtstalker.config file, then you will need to enable browsing of the /usr/local directory. 

Use menu option "Finder : Go : Go to Folder ..."
Then drag "local" to the Finder sidebar.

Start Qtstalker

In a Terminal window, as a normal user, do this:

qtstalker

Where does stuff get installed

See the main notes about this topic.

Mac OS X stores user preferences information at:

~/Library/Preferences/*qtstalker*.plist
