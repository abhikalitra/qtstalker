Qtstalker: Installing on Mac OS X

Requirements

  • Mac OS X Leopard (10.5) or Snow Leopard (10.6)
  • Apple Xcode Tools. To build Qtstalker from sources.

Installation

See the general Installation instructions for overview, requirements, and tips.

There are other build systems and IDEs, however this works from the command-line with minimal changes to Qtstalker sources.

Install TA-Lib

Install TA-Lib 0.4.0 from source. This is easy. (It could instead be installed via MacPorts.) 

./configure
make
sudo make install

This will install TA-Lib at /usr/local ... See important notes about ensuring proper installation and configuration if at a non-standard location.

Install Qt

Install Qt 4.6 or newer from source. (This could instead be installed via MacPorts.)

For Leopard this is straight-forward: 

./configure
make
sudo make install

For Snow Leopard use this configuration: 

./configure -cocoa -arch x86_64
make
sudo make install

See important notes about ensuring proper installation and adding to your $PATH environment variable to ensure that 'qmake' etc can be found.

Install Qwt

Install Qwt 5.2 or newer from source. (It could instead be installed via MacPorts.)

See important notes about ensuring proper installation and adding to your $DYLD_LIBRARY_PATH environment variable to ensure that its shared library can be found.

Building Qtstalker

Prepare for compiling Qtstalker:

edit qtstalker.config ... to add the following lines (using whatever relevant version numbers): 

INCLUDEPATH += /usr/local/Trolltech/Qt-4.6.3/include/Qt
LIBS = -L/usr/local/Trolltech/Qt-4.6.3/lib
INCLUDEPATH += /usr/local/qwt-5.2.1/include
LIBS += -L/usr/local/qwt-5.2.1/lib

For Snow Leopard, edit configure ... to tweak the 'qmake' invocation to include "CONFIG += qt x86_64"

Now compile Qtstalker: 

./configure
make
sudo make install

Now start Qtstalker: 

/usr/local/bin/qtstalker.app/Contents/MacOS/qtstalker

Tip: Use the "Keep in Dock" making it easier to start.

Tip: Use the "Console" application to see any log messages, e.g. during data loading.
