Script API Reference


Quotes


QUOTE_SET,<PLUGIN>,<EXCHANGE>,<SYMBOL>,<DATE_FORMAT>,<?>

This function is used to save quotes into the database.
  • <PLUGIN> is the quote type to use (Stock, Futures).
  • <EXCHANGE> is the exchange the quote originates from. ie. 'New York Stock Exchange' See here for a list of exchanges used.
  • <SYMBOL> is the security symbol ie. GOOG
  • <DATE_FORMAT> is the date format. See here for the date format structure.
  • <?> This is the quote data separated by commas. You may put as many quotes together as desired as long as they belong to the same symbol. The format is determined by the <PLUGIN> used are:
    • Stock - Date,Open,High,Low,Close,Volume
    • Futures - Date,Open,High,Low,Close,Volume,OI

For example. If we wanted to save 2 quotes for a symbol, the text would be:

"QUOTE_SET,Stock,NASDAQ,MSFT,yyyy-MM-dd HH:MM:SS,2010-03-09 13:24:00,25.50,27.25,23.75,26.25,123456789,2010-03-10 09:59:00,26.50,28.25,24.75,27.25,123456789"


On success a '0' is returned. On failure, a '1' is returned.



QUOTE_SET_NAME,<EXCHANGE>,<SYMBOL>,<NAME>


This function will set the company name of the symbol.
  • <EXCHANGE> is the exchange the symbol trades on. ie. 'NASDAQ'
  • <SYMBOL> is the symbol. ie. 'MSFT'
  • <NAME> is the company name to use. ie. 'Microsoft Corporation'

For example. Setting the company name for MSFT would be:

"QUOTE_SET_NAME,NASDAQ,MSFT,Microsoft Corporation"


 On success a '0' is returned. On failure, a '1' is returned.



Symbols


SYMBOL_GET

This function returns the currently loaded symbol in qtstalker to the script. On success the symbol is returned. On failure, a '1' is returned.



SYMBOL_LIST,<EXCHANGE>,<SEARCH_STRING>

This function allow scripts to return a list of symbols in the quote database. On success a csv list of symbols is returned. On failure, the string "ERROR" is returned. If both fields are left blank, then all symbols in the database will be retrurned.

  • <EXCHANGE> Specifies a particular exchange to search on. If left blank, exchange field will be ignored and all symbols from all exchanges that match <SEARCH_STRING> will be returned.
  • <SEARCH_STRING> Is the search string pattern. The % is used to match any length. ie %IBM% would return any symbols containing IBM. The _ is used to match on a single character. If left blank, then all symbols that match the <EXCHANGE> field will be returned.


Utilities


CLEAR

     Used in script loops to clear memory of data used in the script. This is MANDATORY when looping in scripts. If not used, we could develop memory leaks or duplicate variable errors. On success a '0' is returned. On failure, a '1' is returned.


Group Functions



GROUP_ADD,<GROUP_NAME>,<GROUP_ITEM>

     This function will add an item to a Qtstalker group. If <GROUP_NAME> does not exist, it will be created and the <GROUP_ITEM> added. On success a '0' is returned. On failure, a '1' is returned.

  • <GROUP_NAME> is the name of the group to add items to.
  • <GROUP_ITEM> is the item we want to add to the group.



GROUP_DELETE,<GROUP_NAME>

     This function will delete a Qtstalker group and all of its contents. On success a '0' is returned. On failure, a '1' is returned.

  • <GROUP_NAME> is the name of the group to delete.



GROUP_GET,<GROUP_NAME>

     This function will return a csv list of <GROUP_NAME>. On success a csv list of the group contents is returned. On failure, the string "ERROR" is returned.

  • <GROUP_NAME> is the name of the group.



Data Exchange

These functions allow indicator scripts to exchange raw data between scripts and Qtstalker. If we want access to raw indicator data we use the INDICATOR_GET function. To create a custom indicator with values created within the script, we use the INDICATOR_SET function


INDICATOR_GET,<NAME>,<BARS>

Request indicator values.

  • NAME is the variable name of the indicator previously created in the script.
  • BARS is the number of bars desired. 0 will return all bars. ie. 20 would return the last 20 bars.

On success, a CSV (comma separated values) string is returned of the entire indicator. Failure returns the string "ERROR". We return "ERROR" instead of "1" because the script might confuse the error code with actual data, so we return letters instead of numbers in this case.




INDICATOR_GET_INDEX,<NAME>,<OFFSET>

Return a single value in a data array

  • NAME is the variable name of the indicator previously created in the script.
  • OFFSET is the index position starting from the last value moving towards the start. ie. 0 would return the last value. 1 would return the 2nd last value etc.
On success, a single value is returned. Failure returns the string "ERROR". We return "ERROR" instead of "1" because the script might confuse the error code with actual data, so we return letters instead of numbers in this case.



INDICATOR_GET_SIZE,<NAME>

Returns the size of a data array.

  • NAME is the variable name of the indicator previously created in the script.

On success, a single value is returned. Failure returns the string "ERROR". We return "ERROR" instead of "1" because the script might confuse the error code with actual data, so we return letters instead of numbers in this case.




INDICATOR_SET,<NAME>,<STRING>

Send data from the script to create a new indicator

  • NAME is the variable name of the dataset.
  • STRING is a CSV string of all values.

On success returns "0", failure returns "1"


Plot


PLOT,<NAME>,<LABEL>,<PLOT_TYPE>

This function allows the indicator script to plot indicators.


  • PLOT is the command.
  • NAME is the indicator previously created to plot.
  • LABEL is the string used to identify the plot on the chart.
  • PLOT_TYPE is the type of plot desired.


The available values of PLOT_TYPE are:

  • Dot
  • Dash
  • Histogram
  • Histogram Bar
  • Line
  • Invisible
  • Horizontal
  • Bar
  • Candle


After the PLOT command string is given, the script must then wait for the return code. The return code will be either "0" for success or "1" for error. If an error occurs, the script should exit execution. Please browse the included scripts for examples of how they should be used.


Indicator

Creates an indicator.

<NAME> is the name given to the indicator

<INPUT> is the input data. This can be either a previously created indicator or bar data as follows:

  • Open - open value of the bar
  • High - high value of the bar
  • Low - low value of the bar
  • Close - close value of the bar
  • Volume - volume value of the bar
  • OI - open interest value of the bar
  • Average Price - (Open + High + Low + Close) / 4 
  • Median Price - (High + Low) / 2
  • Typical Price - (High + Low + Close) / 3
  • WCPrice - (High + Low + (Close * 2)) / 4

<OPERATOR> is a comparison operator. They are:

  • "=" equal to
  • "<" less than
  • "<=" less than or equal to
  • ">" greater than
  • ">=" greater than or equal to

COLOR

INDICATOR,COLOR,All,<INPUT>,<COLOR>

INDICATOR,COLOR,Compare,<INPUT>,<INPUT_2>,<VALUE>,<COLOR>


This function has 2 methods. The 'All' method will color all bars with the same color. The 'Compare' method compares the values of <INPUT> data array with the number <VALUE>. If they are equal then <COLOR> is applied to the <INPUT_2> data array.


All method:
  • <INPUT> is the array to color.
  • <COLOR> is the color to use. Either a RGB value (#FF0000) or english word (red) can be used.

The Compare method:
  • <INPUT> is the line that determines the color
  • <INPUT_2> is the line that receives the color
  • <VALUE> is the number compared in <INPUT>. If <VALUE> and <INPUT> are equal, then <COLOR> is applied to <INPUT_2>
  • <COLOR> is the color name that is applied to <INPUT_2>. The color can either be an RGB value (#FF0000) or english word (red).

COMPARE

INDICATOR,COMPARE,<NAME>,<INPUT_1>,<INPUT_2>,<OPERATOR>


Compares  <INPUT_1> with <INPUT_2> using the <OPERATOR> for comparison. <INPUT_1> and <INPUT_2> can either be a data array or a numerical value. The result is a boolean array of TRUE/FALSE values. The comparison order is made <INPUT_1> <OPERATOR> <INPUT_2>.
