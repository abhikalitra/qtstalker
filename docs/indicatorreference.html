Script API Reference


Symbols

This function allow scripts to return a list of symbols in the quote database.

SYMBOL_LIST,SEARCH_STRING
  • SEARCH_STRING is the search string pattern. A '*' will return all symbols in the database. The % is used to match any length. ie %IBM% would return any symbols containing IBM. The _ is used to match on a single character.


Data Exchange

These functions allow indicator scripts to exchange raw data between scripts and Qtstalker. If we want access to raw indicator data we use the GET_INDICATOR function. To create a custom indicator with values created within the script, we use the SET_INDICATOR function

To request indicator values:

GET_INDICATOR,NAME,BARS

  • NAME is the variable name of the indicator previously created in the script.
  • BARS is the number of bars desired. 0 will return all bars. ie. 20 would return the last 20 bars.

On success, a CSV (comma separated values) string is returned of the entire indicator. Failure returns the string "ERROR". We return "ERROR" instead of "1" because the script might confuse the error code with actual data, so we return letters instead of numbers in this case.

To send data from the script to create a new indicator:

SET_INDICATOR,NAME,STRING

  • NAME is the variable name of the dataset.
  • STRING is a CSV string of all values.

On success returns "0", failure returns "1";

Plot Format

This function allows the indicator script to plot indicators.

The plot format is:

PLOT,NAME,LABEL,COLOR,PLOT_TYPE

  • PLOT is the command.
  • NAME is the indicator previously created to plot.
  • LABEL is the string used to identify the plot on the chart.
  • COLOR is the color of the plot.
  • PLOT_TYPE is the type of plot desired.

The available values of PLOT_TYPE are:

  • Dot
  • Dash
  • Histogram
  • Histogram Bar
  • Line
  • Invisible
  • Horizontal
  • Bar
  • Candle

After the PLOT command string is given, the script must then wait for the return code. The return code will be either "0" for success or "1" for error. If an error occurs, the script should exit execution. Please browse the included scripts for examples of how they should be used.

Indicator Format

<NAME> is the name given to the indicator

<INPUT> is the input data. This can be either a previously created indicator or bar data as follows:

  • "Open" the open value of the bar
  • "High" the high value of the bar
  • "Low" the low value of the bar
  • "Close" the close value of the bar
  • "Volume" the volume value of the bar
  • "OI" the open interest value of the bar

<OPERATOR> is a comparison operator. They are:

  • "=" equal to
  • "<" less than
  • "<=" less than or equal to
  • ">" greater than
  • ">=" greater than or equal to
  • "AND" logical AND
  • "OR" logical OR

COLOR

INDICATOR,COLOR,<INPUT>,<INPUT_2>,<VALUE>,<COLOR>

This function compares the values of <INPUT> data array with the number <VALUE>. If they are equal then

<COLOR> is applied to the <INPUT_2> data array.

<INPUT> is the line that determines the color

<INPUT_2> is the line that receives the color

<VALUE> is the number compared in <INPUT>. If <VALUE> and <INPUT> are equal, then <COLOR> is applied to <INPUT_2>

<COLOR> is the color name that is applied to <INPUT_2>


COMPARE

INDICATOR,COMPARE,<NAME>,<INPUT_1>,<INPUT_2>,<OPERATOR>

This function compares the values of <INPUT_1> data array with <INPUT_2> data array using the <OPERATOR> for comparison.

The result is a boolean array of TRUE/FALSE values. The comparison order is made <INPUT_1> <OPERATOR> <INPUT_2>.


COMPARE2

INDICATOR,COMPARE2,<NAME>,<INPUT>,<VALUE>,<OPERATOR>

This function compares the values of <INPUT> data array with the number <VALUE> using the <OPERATOR> for comparison.

The result is a boolean array of TRUE/FALSE values. The comparison order is made <INPUT> <OPERATOR> <VALUE>.
