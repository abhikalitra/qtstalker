Script API - Futures Quotes


QUOTE,Futures,SET_QUOTE,<EXCHANGE>,<SYMBOL>,<DATE FORMAT>,<DATE>,<OPEN>,<HIGH>,<LOW>,<CLOSE>,<VOLUME>,<OPEN INTEREST>

This script command is used to save futures quotes into the database.
  • <EXCHANGE> is the exchange the quote originates from. ie. 'FCBT'
  • <SYMBOL> is the futures symbol ie. 'C_2010_Z'
  • <DATE FORMAT> is the date format.
  • <DATE> this is the date value that conforms to the <DATE FORMAT> format
  • <OPEN> open value
  • <HIGH> high value
  • <LOW> low value
  • <CLOSE> close value
  • <VOLUME> volume value
  • <OPEN INTEREST> open interest value

For example. If we wanted to save a futures quote, the text would be:

"QUOTE,Futures,SET_QUOTE,FCBT,C_2010_Z,yyyy-MM-dd HH:mm:ss,2010-03-09 13:24:00,25.50,27.25,23.75,26.25,123456789,123456"

On success a '0' is returned. On failure, 'ERROR' is returned.


QUOTE,Futures,SAVE_QUOTES

This function will save quotes to the database after "SET_QUOTE" commands have been previously run in the script. If this command is not used after quote data has been "SET_QUOTE", then the quote data will be lost and NOT saved to the database when the script completes.

Tip: This command should be used once after all quotes within a running script have been added. You will get a 1000% speed increase.

On success a '0' is returned. On failure, 'ERROR' is returned.




QUOTE,Futures,SET_NAME,<EXCHANGE>,<SYMBOL>,<NAME>

This function will set the name of the futures symbol.
  • <EXCHANGE> is the exchange the symbol trades on. ie. 'FCBT'
  • <SYMBOL> is the futures symbol. ie. 'C_2010_Z'
  • <NAME> is the name to use. ie. 'Corn 2010 December'

For example. Setting the name for C_2010_Z would be:

"QUOTE,Futures,SET_NAME,FCBT,C_2010_Z,Corn 2010 December"


 On success a '0' is returned. On failure, 'ERROR' is returned.




QUOTE,Futures,SET_CODE,<EXCHANGE>,<SYMBOL>,<CODE>

This function will set the exchange futures code. A futures code is the commodity exchange code. Corn would be 'C' or S&P 500 index would be 'SP'.
  • <EXCHANGE> is the exchange the symbol trades on. ie. 'FCME'
  • <SYMBOL> is the futures symbol. ie. 'SP_2010_Z'
  • <CODE> is the exchange code to use. ie. 'SP' for S&P 500 index futures

For example. Setting the code for SP_2010_Z would be:

"QUOTE,Futures,SET_CODE,FCME,SP_2010_Z,SP"

On success a '0' is returned. On failure, 'ERROR' is returned.




QUOTE,Futures,SET_MONTH,<EXCHANGE>,<SYMBOL>,<MONTH>

This function will set the futures expiry month code of the futures symbol.
  • <EXCHANGE> is the exchange the symbol trades on. ie. 'FCME'
  • <SYMBOL> is the futures symbol. ie. 'SP_2010_Z'
  • <MONTH> is the expiry month code to use. ie. 'Z'  Month codes are:
    • F - January
    • G - Febuary
    • H - March
    • J - April
    • K - May
    • M - June
    • N - July
    • Q - August
    • U - September
    • V - October
    • X - November
    • Z - December
For example. Setting the expiry month code for SP_2010_Z would be:

"QUOTE,Futures,SET_MONTH,FCME,SP_2010_Z,Z"

On success a '0' is returned. On failure, 'ERROR' is returned.




QUOTE,Futures,SET_YEAR,<EXCHANGE>,<SYMBOL>,<YEAR>

This function will set the futures expiry year.
  • <EXCHANGE> is the exchange the symbol trades on. ie. 'FCME'
  • <SYMBOL> is the futures symbol. ie. 'SP_2010_Z'
  • <YEAR> is a 4 digit integer value for the expiry year. ie. '2010'

For example. Setting the expiry year for futures symbol 'SP_2010_Z' would be:

"QUOTE,Futures,SET_YEAR,FCME,SP_2010_Z,2010"

On success a '0' is returned. On failure, 'ERROR' is returned.




QUOTE,Futures,DELETE,<EXCHANGE>,<SYMBOL>

This function will permanently delete the symbol from the quotes database.
  • <EXCHANGE> is the exchange the symbol trades on. ie. 'FCME'
  • <SYMBOL> is the futures symbol. ie. 'SP_2010_Z'

For example. Deleting the 'SP_2010_Z' would be:

"QUOTE,Futures,DELETE,FCME,SP_2010_Z"

On success a '0' is returned. On failure, 'ERROR' is returned.



QUOTE,Futures,GET_QUOTES,<NAME>,<EXCHANGE>,<SYMBOL>,<BAR FIELD>,<BAR LENGTH>,<BARS>

This function will create an array of values using the bar field of the symbol requested. These values can be manipulated by the various INDICATOR group of script commands.

  • <NAME> is the variable given to the array to create
  • <EXCHANGE> is an exchange
  • <SYMBOL> is the quote symbol name
  • <BAR FIELD> is the bar field:
    • Open
    • High
    • Low
    • Close
    • Volume
    • OI
  • <BAR LENGTH> is the bar length:
    • 1 Minute
    • 5 Minute
    • 10 Minute
    • 15 Minute
    • 30 Minute
    • 60 Minute
    • Daily
    • Weekly
    • Monthly
  • <BARS> is the number of bars to create

For example, if we wanted to create an array of daily close values from the SP_2010_Z symbol, we would issue the command:

"QUOTE,Futures,GET_QUOTES,close_array,FCME,SP_2010_Z,Close,Daily,275"

On success a "0" is returned. On failure, "ERROR" is returned.




QUOTE,Futures,RENAME,<OLD EXCHANGE>,<OLD SYMBOL>,<NEW EXCHANGE>,<NEW SYMBOL>

This function will rename an existing symbol in the quotes database.
  • <OLD EXCHANGE> is the exchange of the symbol to be renamed e.g. 'FCME'
  • <OLD SYMBOL> is the futures symbol to be renamed. e.g. 'SP_2010_Z'
  • <NEW EXCHANGE> is the new exchange
  • <NEW SYMBOL> is the new symbol

On success a '0' is returned. On failure, 'ERROR' is returned.
