Chande Momentum Oscillator - CMO


Parameters:

  • Period - The CMO period
  • Color - plot color
  • Line Type - The type of plot to draw
  • Label - The text to identify the plot
  • Adaptive CMO - Option to use adaptive method.
  • Min Lookback
  • Max Lookback
  • Buy Zone Color
  • Sell Zone Color
  • Buy Zone - The buy zone value
  • Sell Zone - The sell zone value

Description:

The Chande Momentum Oscillator (CMO) was developed by Tushar Chande. Chande developed the CMO to  capture what he calls “pure momentum.” For more information on the CMO and other indicators see  the The New Technical Trader by Tushar Chande and Stanley Kroll. The CMO is closely related to other  momentum oriented indicators such as RSI, Stochastic, Rate-of-Change, etc. It is most closely related to  Welles Wilder’s RSI (see Relative Strength Index), yet it differs in several ways:

* It uses data for both up days and down days in the numerator, thereby directly measuring momentum.
* The calculations are applied on unsmoothed data. Therefore, short-term extreme movements in price are not hidden. Once calculated, smoothing can be applied to the CMO, if desired.
* The scale is bounded between +100 and -100. allowing you to clearly see changes in net momentum  using the 0 level. The bounded scale also allows comparisons between different securities.

The CMO can be used in the usual ways. Overbought/oversold. The primary method of interpreting the  CMO islooking for extreme overbought and oversold conditions. Chande quantifies an overbought level at +50 and the oversold level at -50. At +50, up-day momentum is three times the down-day momentum. Like  wise, at -50, down-day momentum is three times the up-day momentum. These levels correspond to the  70/30 levels on the RSI indicator.

Trendiness. The can also be used to measure trendiness. The higher the CMO, the stronger the trend.  Low values of the CMO show a security in a sideways trading range.

There are two modes for this indicator Standard and Adaptive. Standard (Adaptive not checked) computes the standard CMO using the period selected by the user. Adaptive (Adaptive checked) first computes the standard deviation of the input time series computed over the period selected by the user. It then uses that standard deviation to modify the lookback period used to compute the CMO between Min Loopback and Max Loopback as selected by the user.

Formula Reference:

Syntax
 CMO(ARRAY INPUT, Period, Adaptive CMO, Min Lookback, Max Lookback)
Function
 ARRAY INPUT can be either an array of bars or a formula variable array of data. Adaptive CMO is either TRUE or FALSE.
Example
 CMO(Close, 14, FALSE, 7, 10)
