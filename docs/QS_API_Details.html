QuoteServer API - Details


These script commands create and modify specific details related to a symbol in the database. There are a few defined detail types and 15 text fields and 15 decimal fields. The text fields are called T1...T15. The decimal fields are called D1..D15. It is up to the user to decide what data goes in them. This allows maximum flexibility and allows field based sql searches to be done. The use of details are optional.

Here are the available <DETAIL> values:

  • Name - used for company name / description text
  • Sector - used by stocks for business type eg. Industrial
  • FuturesCode - exchange code used by futures contracts to describe the futures type eg. 'C' for corn.
  • FuturesExpMonth - the futures contract expiry month eg. 'Z' for december
  • FuturesExpYear - the futures contract expiry year eg. '2012'
  • T1...T15 - user defined text fields
  • D1...D15 - user defined decimal fields




Details,G,<EXCHANGE>,<SYMBOL>,<DETAILS>\n

Get (return) 1 or more details for a specific database symbol.

  • <EXCHANGE> is the exchange the quote originates from. eg. 'XNYS'
  • <SYMBOL> is the symbol eg. 'GOOG'
  • <DETAILS> is the detail type. this is a colon delimited string containing one or more detail types eg. 'T1:T2'
On success, the details are returned in a colon delimited string in the same order as requested.

For example. Say we used T1 to store the company name and T2 to store the business type. We write the string 'T1:T2'. The returned text would be 'Google Corp:Internet'.

If there are no details in the database, an empty string is returned. Failure returns 'ERROR\n'.

A full example would be 'Details,G,XNYS,GOOG,T1:T2\n'




Details,S,<EXCHANGE>,<SYMBOL>,<DETAILS>,<DETAILS TEXT>\n

Set (save) 1 or more details for a specific database symbol.

  • <EXCHANGE> is the exchange the quote originates from. eg. 'XNYS'
  • <SYMBOL> is the symbol eg. 'GOOG'
  • <DETAILS> is the detail type. this is a colon delimited string containing one or more detail types eg. 'T1:T2'
  • <DETAILS TEXT> the actual data text to store. this is a colon delimited string that will be stored in the same order the <DETAILS> contains.

For example. Say we used T1 to store the company name and T2 to store the business type. We write the string 'T1:T2' for the <DETAILS> portion. The <DETAILS TEXT> portion would be 'Google Corp:Internet'.

On success, 'OK\n' is returned. Failure returns 'ERROR\n'.

A Full example would be 'Details,S,XNYS,GOOG,T1:T2,Google Corp:Internet\n'



Details,R,<EXCHANGE>,<SYMBOL>\n

Removes all details related to the given symbol in the database.

  • <EXCHANGE> is the exchange the quote originates from. eg. 'XNYS'
  • <SYMBOL> is the symbol eg. 'GOOG'
On success, 'OK\n' is returned. Failure returns 'ERROR\n'.

A Full example would be 'Details,R,XNYS,GOOG\n'
