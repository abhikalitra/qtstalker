Qtstalker: Documentation

Viewing the documentation

Local documentation can be viewed with a web browser, by opening /usr/local/share/doc/qtstalker/html/index.html and can be edited at path-to-your-source/qtstalker/docs/

Source code documentation can be viewed with a web browser, by opening path-to-your-source/qtstalker/docs/api/index.html (If that documentation has not yet been generated using Doxygen, then see the notes below.)

Documentation can also be viewed via the Qt Assistant application, by selecting from the Qtstalker "Help" menu.

Using Doxygen to generate documentation

Documentation for all of the Qtstalker source code can be generated. Any comment blocks in the source code are added to the documentation.

This documentation generation system can also be used to document the source code of your own indicators and other scripts.

Requirements

Install Doxygen and ensure that the 'doxygen' executables are available on your $PATH

Optionally install Graphviz to generate better diagrams. Set HAVE_DOT=YES in the qtstalker/misc/doxygen/config.txt file.

Optionally install DoxygenFilter (or another filter) to enable processing of Perl scripts.

Configuration

The configuration file for Doxygen is located at: qtstalker/misc/doxygen/config.txt

By default it will process C++ and Python files. With the abovementioned filter it will do Perl scripts and others.

To process your set of user indicators and other scripts, configure the INPUT parameter of the config.txt to be: 

INPUT = . /Users/me/Documents/work/qtstalker/scripts/

Note: Be sure to retain the configuration for the path "." (as shown above) to process all of the Qtstalker source files.

See the Doxygen instructions for documenting your source code.

There is one example for the Qtstalker Disparity indicator. Note: Trying to keep documentation simple.

Processing

In the top-level directory of the Qtstalker distribution, do './build-api-docs.sh' and then do the normal Qtstalker build and install process 'make distclean; ./configure' etc.
