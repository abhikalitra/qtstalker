Script API - Indicators

These CUS commands allow creation and manipulation of indicator values.

NEW

INDICATOR,NEW,<METHOD>,<NAME>

Create a new indicator. The <METHOD> value determines if the new indicator is empty or filled with values.

  • <METHOD> is one of the following values:
    • EMPTY - no default values
    • Open - filled with open bar values from the currently loaded chart
    • High - filled with high bar values from the currently loaded chart
    • Low - filled with low bar values from the currently loaded chart
    • Close - filled with close bar values from the currently loaded chart
    • Volume - filled with volume bar values from the currently loaded chart
    • OI - filled with open interest bar values from the currently loaded chart
    • AvgPrice - filled with average price values from the currently loaded chart
    • MedianPrice - filled with median price values from the currently loaded chart
    • TypicalPrice - filled with typical price values from the currently loaded chart
    • WCPrice - filled with weighted close price values from the currently loaded chart
  • <NAME> is the variable name to create

On success, a '0' is returned. Failure returns 'ERROR'.

GET_INDEX

INDICATOR,GET_INDEX,<NAME>,<INDEX>

Return a single value of an indicator at index position.

  • <NAME> is the variable name of the indicator previously created in the script
  • <INDEX> is the index position

On success, a single value is returned. Failure returns the string "ERROR".

GET_INDEX_DATE

INDICATOR,GET_INDEX_DATE,<INDEX>

Returns the Qt::ISODate at index position. The date is yyyy-MM-ddTHH:mm:ss format.

  • <INDEX> is the index position

On success, a single value is returned. Failure returns the string "ERROR".

GET_RANGE

INDICATOR,GET_RANGE,<NAME>

Returns the start and end index values of an indicator.

  • <NAME> is the variable name of the indicator previously created in the script.

On success, the start and end index values is returned. Failure returns the string "ERROR". The values are separated by a comma.

e.g. "0,100" would be returned indicating the first index position is 0 and the last is 100.

SET_INDEX

INDICATOR,SET_INDEX,<NAME>,<INDEX>,<VALUE>,<COLOR>

Set the bar value and color at index position. If the index position is empty, the values will be inserted. Otherwise, the values will be replaced.

  • <NAME> is the variable name of the dataset.
  • <INDEX> is the index position
  • <VALUE> is the value
  • <COLOR> is the color of the bar

On success returns '0', failure returns "ERROR"

SET_COLOR

INDICATOR,SET_COLOR,<NAME>,<INDEX>,<COLOR>

Set the bar color at index position. If the index position is empty, the command will fail. Otherwise, the color value will be replaced.

  • <NAME> is the variable name of the dataset.
  • <INDEX> is the index position
  • <COLOR> is the color of the bar

On success returns '0', failure returns "ERROR"

DELETE

INDICATOR,DELETE,<NAME>

Deletes the indicator from memory. Used in script loops where an indicator is created with multiple symbols.

  • <NAME> is the variable name of the indicator

On success returns '0', failure returns "ERROR"

SET_COLOR_ALL

INDICATOR,SET_COLOR_ALL,<NAME>,<COLOR>

Sets all bars to <COLOR>.

  • <NAME> is the variable name of the dataset.
  • <COLOR> is the color of the bar

On success returns '0', failure returns "ERROR"

SET_PLOT_STYLE

INDICATOR,SET_PLOT_STYLE,<NAME>,<PLOT STYLE>

Sets the indicator to the desired plotting style.

  • <NAME> is the variable name of the dataset.
  • <PLOT STYLE> the plot style

On success returns '0', failure returns "ERROR"

SET_PLOT

INDICATOR,SET_PLOT,<NAME>,<Z>

Plots the indicator on a chart in the specified order. If we have multiple plots, the order in which they are overlayed becomes essential.

  • <NAME> is the variable name of the dataset.
  • <Z> the order in which plots are drawn. the plots are drawn in ascending order. eg. 0 first, 100 last.

On success returns '0', failure returns "ERROR"
